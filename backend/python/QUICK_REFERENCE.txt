================================================================================
WHAT HAPPENS IN THE PIPELINE:
================================================================================

Step 1: PDF → Word
   • Measures PDF page size
   • Creates Word doc with exact page dimensions
   • Adds 0.5" margins on all sides
   • Preserves table structure
   • Takes ~90 seconds per 22-page PDF

Step 2: Word → Excel
   • Extracts all tables from Word
   • Converts text numbers to actual numbers
   • Adds professional formatting:
     - Borders on all cells
     - Right-aligned numbers (2 decimals)
     - Left-aligned text
     - 18-unit column widths
   • Takes ~5 seconds
   • Deletes temporary Word file

Result: Professional Excel with exact PDF structure

================================================================================
PIPELINE BENEFITS OVER DIRECT PDF→EXCEL:
================================================================================

Why use this 2-step pipeline?

1. Exact Layout Preservation
   - Direct PDF→Excel loses page structure
   - Word acts as intermediate preserving layout
   - Word dimensions match PDF exactly

2. Better Table Extraction
   - Word→Excel preserves table structure better
   - Direct extraction often misses table boundaries
   - Results in properly aligned columns

3. Proper Number Handling
   - Text numbers detected and converted
   - Decimal formatting applied (2 places)
   - Proper alignment (right for numbers, left for text)

4. Professional Output
   - Borders applied to all cells
   - Column widths optimized
   - Consistent formatting throughout
   - No text wrapping (clean appearance)

5. More Reliable
   - Uses pdf2docx which is layout-aware
   - Temporary Word file acts as validation
   - Better error detection

Result: Excel that looks professional and is usable for data analysis

================================================================================
TROUBLESHOOTING:
================================================================================

Q: Why is conversion slow?
A: pdf2docx takes time for layout analysis (~90s). This is normal.

Q: My numbers still look like text?
A: The function tries to convert numeric values. Check if they have
   non-numeric characters that can't be converted.

Q: Output file is blank?
A: Check if input PDF has tables. The pipeline extracts Word tables.

Q: Can I use this with non-English PDFs?
A: Yes, the pipeline handles any language that pdf2docx supports.

Q: Can I convert multiple PDFs at once?
A: Yes, use batch mode: python ... -b -o output_directory
